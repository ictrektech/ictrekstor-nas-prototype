# PRD 功能实现约束

> 本文档按**开发阶段**记录各 PRD 定义的功能实现约束，说明在当前阶段哪些功能已实现、哪些暂未实现、以及本阶段的实现预期。
>
> PRD 描述的是最终功能预期，本文件用于限定当前开发阶段的实际实现边界。
>
> **说明**："开发阶段"是逻辑上的划分，可能与一个或多个实际迭代版本关联，后续会持续更新阶段与版本号的对应关系。

---

## 约束说明格式

每条约束包含以下信息：

- **所属 PRD / 功能模块**：对应 PRD 中的功能模块
- **约束类型**：`暂不实现` / `部分实现` / `占位实现` / `默认状态` / `简化实现`
- **约束描述**：当前阶段的实际处理方式和原因
- **后续计划**：预计在哪一阶段完整实现（如有）

---

## 开发阶段一（当前阶段）

> 本阶段聚焦核心模块的原型搭建，包括文件管理、存储管理、系统设置中的设备/用户/存储空间管理，以及 AI 助手的基础功能。
>
> **关联迭代版本**：待补充

### 系统设置 (`/system/*`)

#### 服务设置页面 — 整体隐藏

- **所属 PRD / 功能模块**：[服务设置](./system/services.md)
- **约束类型**：`暂不实现`
- **约束描述**：
  - 鉴于当前 SMB 管理功能暂不实现，服务设置页面已无实际可用功能，本阶段**整体隐藏该页面**（不显示在菜单中，不开放路由访问）。
  - SMB 服务在系统中保持**默认启用**状态，由系统底层自行维护，不在前端提供任何管理入口。
- **原因**：本阶段优先聚焦存储空间、用户管理等核心模块，服务管理功能放到后续阶段统一实现。
- **后续计划**：开发阶段二或更高阶段，完整实现服务设置页面，包括 SMB 参数配置、各服务启停管理、快捷信息展示等功能。

### 首页 (`/dashboard`)

#### 首页 — 不计划维护

- **所属 PRD / 功能模块**：[首页 / Dashboard](./home/index.md)
- **约束类型**：`暂不实现`
- **约束描述**：
  - 首页（Dashboard）由平台组负责维护，不属于存储需求，**不计划在本项目中维护**。
  - 本阶段不实现首页相关的原型页面和交互。
- **后续计划**：由平台组在独立项目中实现和维护。

### 存储管理 (`/storage/*`)

#### 磁盘详情 — 优先级低

- **所属 PRD / 功能模块**：[磁盘详情](./storage/disk-detail.md)
- **约束类型**：`暂不实现`
- **约束描述**：
  - 磁盘详情属于扩展功能，不是核心功能，且开发成本较高，**本阶段实现优先级低**。
  - 当前阶段暂不实现单块磁盘的完整信息及 SMART 监控页面。
- **后续计划**：根据产品优先级评估，在后续阶段决定是否实现。

#### 存储池详情 — 优先级低

- **所属 PRD / 功能模块**：[存储池详情](./storage/pool-detail.md)
- **约束类型**：`暂不实现`
- **约束描述**：
  - 存储池详情属于扩展功能，**本阶段实现优先级低**。
  - 当前阶段暂不实现单个存储池的完整信息查看页面。
- **后续计划**：根据产品优先级评估，在后续阶段决定是否实现。

### AI 助手 (`/ai/*`)

#### 智能对话 — 不计划实现

- **所属 PRD / 功能模块**：[智能对话 / AI Chat](./ai/chat.md)
- **约束类型**：`暂不实现`
- **约束描述**：
  - AI 相关功能不属于存储需求，**不计划在本项目中实现**。
  - 本阶段不实现 AI 助手相关的原型页面和交互。
- **后续计划**：暂无计划在本项目中实现。

### 其他模块

- **文件管理 (`/file/*`)**：本阶段按对应 PRD 完整实现，暂无约束。
