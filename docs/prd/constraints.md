# PRD 功能实现约束

> 本文档按**开发阶段**记录各 PRD 定义的功能实现约束，说明在当前阶段哪些功能已实现、哪些暂未实现、以及本阶段的实现预期。
>
> PRD 描述的是最终功能预期，本文件用于限定当前开发阶段的实际实现边界。
>
> **说明**："开发阶段"是逻辑上的划分，可能与一个或多个实际迭代版本关联，后续会持续更新阶段与版本号的对应关系。

---

## 约束说明格式

每条约束包含以下信息：

- **所属 PRD / 功能模块**：对应 PRD 中的功能模块
- **约束类型**：`暂不实现` / `部分实现` / `占位实现` / `默认状态` / `简化实现`
- **约束描述**：当前阶段的实际处理方式和原因
- **后续计划**：预计在哪一阶段完整实现（如有）

---

## 开发阶段一（当前阶段）

> 本阶段聚焦核心模块的原型搭建，包括文件管理、存储管理、系统设置中的设备/用户/存储空间管理，以及 AI 助手的基础功能。
>
> **关联迭代版本**：待补充

### 系统设置 (`/system/*`)

#### 服务设置页面 — 整体隐藏

- **所属 PRD / 功能模块**：[服务设置](./system/services.md)
- **约束类型**：`暂不实现`
- **约束描述**：
  - 鉴于当前 SMB 管理功能暂不实现，服务设置页面已无实际可用功能，本阶段**整体隐藏该页面**（不显示在菜单中，不开放路由访问）。
  - SMB 服务在系统中保持**默认启用**状态，由系统底层自行维护，不在前端提供任何管理入口。
- **原因**：本阶段优先聚焦存储空间、用户管理等核心模块，服务管理功能放到后续阶段统一实现。
- **后续计划**：开发阶段二或更高阶段，完整实现服务设置页面，包括 SMB 参数配置、各服务启停管理、快捷信息展示等功能。

#### 设备管理 — 盘位与硬盘联动 — 优先级中

- **所属 PRD / 功能模块**：[设备管理](./system/device.md) / 盘位与硬盘联动
- **约束类型**：`暂不实现`
- **约束描述**：
  - 盘位与硬盘的双向联动功能（点击盘位高亮硬盘、点击定位按钮高亮盘位等）**本阶段实现优先级中等**。
  - 实现该功能需要先行梳理硬件接口标识，当前基础设施尚不完善。
- **后续计划**：待硬件接口标识梳理完成后，在后续阶段实现。

#### 设备管理 — 磁盘闪烁 — 优先级低

- **约束描述**：
  - 硬盘卡片上的"闪烁"功能（用于物理定位硬盘）**本阶段实现优先级低**。
  - 该功能与硬件强相关，实现成本较高。
- **后续计划**：根据硬件支持情况，在后续阶段决定是否实现。
