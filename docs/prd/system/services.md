# PRD: 服务设置

## 页面概要

| 属性 | 值 |
|------|-----|
| 路由 | `/system/services` |
| 菜单标题 | 服务设置 |
| 菜单图标 | `lucide:server-cog` |
| 页面权限 | 管理员 |
| 页面类型 | 管理页（主页面） |

## 需求概述

**总体功能**：服务设置页面用于管理 NAS 系统对外提供的各类服务，包括文件共享服务（SMB/NFS/AFP）、网络协议服务（FTP/WebDAV）等。页面以卡片形式展示各服务，支持启用/禁用、配置参数、查看使用指南。

**目标用户**：NAS 系统管理员，负责配置系统对外服务能力。

**核心价值**：统一管理 NAS 对外服务，方便管理员按需开启/配置各类文件共享和网络协议服务。

---

## 功能规格

### 功能模块: 服务分类展示

#### 功能: 服务分类 Tab
- **功能描述**：按服务类型对服务进行分组展示。
- **预期行为**：
  - 当前支持的服务分类：
    - **文件共享**：SMB 共享（Windows）、NFS 共享（Linux/Unix）、AFP 共享（macOS）
    - **网络协议**：FTP 服务、WebDAV 服务
  - 以 Tab 或分组形式展示各类服务
  - 支持按服务名称搜索
- **约束条件**：
  - 服务列表由系统预定义，不可动态添加
- **可用场景**：按类别查找和管理服务。
- **涉及接口**：
  - 获取服务列表

---

### 功能模块: 服务卡片

#### 功能: 服务信息卡片
- **功能描述**：以卡片形式展示单个服务的概要信息。
- **预期行为**：
  - 每张卡片展示：服务图标和名称、服务描述
  - 显示启用/禁用开关
  - 显示运行状态标签（运行中/已停止），颜色区分
  - 显示快捷信息：共享路径、端口、认证方式等
  - 提供操作按钮：配置、使用指南
- **约束条件**：
  - 部分服务存在依赖关系（如 SMB 依赖特定系统组件）
  - 服务状态切换可能需要数秒生效
- **可用场景**：快速查看各服务的状态和基本信息。
- **涉及接口**：
  - 获取服务列表（含状态和配置）

---

### 功能模块: 服务配置

#### 功能: 服务参数配置
- **功能描述**：为每个服务配置具体运行参数。
- **预期行为**：
  - 点击"配置"按钮，打开服务配置弹窗
  - 不同服务有不同配置项：
    - **SMB**：工作组、服务器名称、SMB 版本、访客访问（启用/禁用）
    - **NFS**：导出路径、客户端权限（IP/网段、读写权限）
    - **FTP**：端口、被动模式、匿名访问、最大连接数
    - **WebDAV**：端口、认证方式（基本/摘要）、HTTPS 启用
  - 配置项变化即时在卡片快捷信息中反映
  - 保存后提示服务重启
- **约束条件**：
  - 配置错误可能导致服务无法启动
  - 端口冲突时保存失败并提示
  - 部分配置变更需要重启服务生效
- **可用场景**：根据网络环境和安全策略调整服务配置。
- **涉及接口**：
  - 获取服务配置详情（用于回显当前配置）
  - 更新服务配置

---

### 功能模块: 使用指南

#### 功能: 客户端连接指南
- **功能描述**：展示各操作系统/客户端连接该服务的方式。
- **预期行为**：
  - 点击"使用指南"按钮，打开使用指南弹窗
  - 分 Tab 展示各操作系统的连接说明：
    - Windows：网络邻居/映射网络驱动器步骤
    - macOS：Finder 连接服务器步骤
    - Linux：mount 命令示例
    - 手机端：支持的应用和连接方式
  - 提供连接地址（支持一键复制）
  - 提供命令行示例（如 `mount -t nfs`）
- **约束条件**：
  - 指南内容基于当前服务的实际配置生成（如 IP、端口、路径）
  - 服务未启用时提示先启用服务
- **可用场景**：帮助终端用户在各自设备上连接 NAS 服务。
- **涉及接口**：
  - 获取服务详情（获取当前运行状态和配置参数，用于生成指南中的动态连接地址）
  - 获取服务使用指南内容

---

### 功能模块: 启用/禁用服务

#### 功能: 服务状态切换
- **功能描述**：启用或禁用某个服务。
- **预期行为**：
  - 点击卡片上的启用/禁用开关
  - 禁用时弹出确认提示，警告该服务将被停止
  - 状态变更后，系统执行启动/停止服务操作
  - 操作完成后卡片状态标签更新
  - 失败时显示错误原因
- **约束条件**：
  - 禁用服务将导致已连接的客户端断开
  - 部分系统核心服务不可禁用
  - 服务启动失败时保持禁用状态并显示错误
- **可用场景**：按需开启服务，或在维护时临时关闭服务。
- **涉及接口**：
  - 启用服务
  - 禁用服务

## 预期交互

### 场景: 启用 SMB 共享
1. 管理员进入"服务设置"页面
2. 找到 SMB 共享卡片，当前状态为"已停止"
3. 点击开关启用 SMB 服务
4. 服务启动成功，状态变为"运行中"
5. 卡片显示快捷信息：端口 445、访客访问已禁用

### 场景: 配置 NFS 导出
1. 管理员点击 NFS 共享卡片的"配置"
2. 弹窗中添加导出路径 `/volume1/data`
3. 设置客户端权限：192.168.1.0/24 网段可读写
4. 保存配置，系统提示 NFS 服务将重启
5. 重启后配置生效

### 场景: 查看使用指南
1. 管理员点击 SMB 共享的"使用指南"
2. 弹窗显示 Windows、macOS、Linux 的连接步骤
3. Windows 页面显示 `\\192.168.1.100\share` 连接地址
4. 管理员复制地址发送给团队成员

## 数据需求（Mock）

- **服务列表数据**：服务ID、名称、类型、描述、启用状态、运行状态、配置参数（端口、路径、认证方式等）、快捷信息
- **各服务配置项默认值**：不同服务的默认配置参数
- **使用指南内容数据**：各操作系统连接说明、命令示例、截图指引
