# PRD: 服务设置

## 页面概要

| 属性 | 值 |
|------|-----|
| 路由 | `/system/services` |
| 菜单标题 | 服务设置 |
| 菜单图标 | `lucide:server-cog` |
| 页面权限 | 管理员 |
| 页面类型 | 管理页（主页面） |

## 需求概述

**总体功能**：服务设置页面用于管理 NAS 系统对外提供的各类服务。页面以统一的卡片风格展示各服务，支持启用/禁用、查看使用指南。每个服务拥有独立的配置数据结构和配置弹窗，当前版本仅完整支持 SMB 共享服务的参数配置，其他服务仅展示启用/禁用状态和基本信息，配置功能作为后续迭代预留。

**目标用户**：NAS 系统管理员，负责配置系统对外服务能力。

**核心价值**：统一管理 NAS 对外服务，方便管理员按需开启/配置各类文件共享和网络协议服务。

## 功能规格

### 功能模块: 服务分类展示

#### 功能: 服务分类 Tab
- **功能描述**：按服务类型对服务进行分组展示。
- **预期行为**：
  - 当前展示的服务分类：
    - **文件共享**：SMB 共享（Windows）、NFS 共享（Linux/Unix）、AFP 共享（macOS）、FTP 共享、WebDAV
    - **管理系统**：HTTP 管理页面
    - **远程访问**：预留分类（当前暂无服务）
  - 以 Tab 形式展示各类服务
  - 页面顶部显示运行中 / 已启用 / 总服务数量统计
- **约束条件**：
  - 服务列表由系统预定义，不可动态添加
- **可用场景**：按类别查找和管理服务。
- **涉及接口**：
  - [获取服务列表](../interface.md#获取服务列表)

---

### 功能模块: 服务卡片

#### 功能: 服务信息卡片
- **功能描述**：以卡片形式展示单个服务的概要信息。
- **预期行为**：
  - 每张卡片展示：服务图标和名称、服务描述
  - 显示启用/禁用开关（Switch）
  - 显示运行状态标签（运行中/已停止/异常），颜色区分
  - 启用状态下显示快捷配置信息（标签列表形式）
  - 启用状态下提供操作按钮：配置、使用指南
  - 禁用状态下卡片置灰，提示"开启后即可配置和访问"
  - 显示访问地址（如 `\\192.168.1.100\share`），支持一键复制
- **约束条件**：
  - 部分服务存在依赖关系（如 SMB 依赖特定系统组件）
  - 服务状态切换可能需要数秒生效
- **可用场景**：快速查看各服务的状态和基本信息。
- **涉及接口**：
  - [获取服务列表](../interface.md#获取服务列表)

---

### 功能模块: SMB 服务配置（当前唯一支持完整配置的服务）

#### 功能: SMB 参数配置
- **功能描述**：为 SMB 共享服务配置具体运行参数。
- **预期行为**：
  - 点击 SMB 卡片的"配置"按钮，打开服务配置弹窗
  - 弹窗以 Tabs 形式组织配置项：
    - **基本设置**：
      - 工作组（workgroup）：字符串输入，默认 `WORKGROUP`
      - 服务器名称（serverName）：字符串输入，默认 `NAS-SERVER`
      - 绑定接口（bindInterface）：下拉选择，选项包括「全部接口」「LAN1 (192.168.1.0/24)」「LAN2 (10.0.0.0/24)」
    - **高级选项**（Switch 开关组）：
      - 启用 SMB 2.0（enableSmb2）：默认开启
      - 启用 SMB 3.0（enableSmb3）：默认开启
      - 启用 SMB 1.0（enableSmb1）：默认关闭，标注"不推荐"
      - 允许访客访问（guestAccess）：默认关闭
      - 广播服务发现（announceService）：默认开启
  - 保存配置后，SMB 卡片上的快捷信息实时同步更新
  - 保存成功后提示"配置已保存"
- **约束条件**：
  - SMB 1.0 存在已知安全风险，界面需标注"不推荐"
  - 端口 445 为系统固定端口，不可修改
  - 配置错误可能导致 SMB 服务无法启动
- **可用场景**：根据网络环境和安全策略调整 SMB 服务配置。
- **涉及接口**：
  - [配置服务参数](../interface.md#配置服务参数)

#### 功能: SMB 快捷信息展示
- **功能描述**：SMB 服务启用后，在卡片上展示关键配置摘要。
- **预期行为**：
  - 以标签列表形式展示：
    - 共享路径：`/share/smb`
    - 端口：`445`
    - 工作组：当前配置值（如 `WORKGROUP`）
    - SMBv3：已启用 / 已禁用
    - 访客访问：允许 / 禁止
  - 配置变更后自动同步更新
- **约束条件**：
  - 快捷信息仅反映当前配置，不涉及实时连接状态统计
- **可用场景**：快速确认 SMB 服务的当前运行参数。

---

### 功能模块: 其他服务（仅展示状态，配置功能暂不支持）

#### 功能: NFS / AFP / FTP / WebDAV / HTTP 管理 状态展示
- **功能描述**：其他服务以统一卡片风格展示，但仅支持启用/禁用切换和查看使用指南。
- **预期行为**：
  - 卡片结构与 SMB 一致：图标、名称、描述、状态标签、启用开关
  - 启用后展示预设的快捷信息（静态标签，不可通过配置弹窗修改）
  - 点击"配置"按钮时，可打开配置弹窗展示默认表单（占位实现），但保存后不做实际生效处理，或提示"该服务配置功能即将上线"
  - 使用指南弹窗正常可用
- **约束条件**：
  - 这些服务的配置弹窗为原型占位，实际配置功能尚未实现
- **可用场景**：了解系统中存在哪些服务及其基本运行状态。
- **涉及接口**：
  - [获取服务列表](../interface.md#获取服务列表)
  - [启动/停止服务](../interface.md#启动停止服务)

---

### 功能模块: 使用指南

#### 功能: 客户端连接指南
- **功能描述**：展示各操作系统/客户端连接该服务的方式。
- **预期行为**：
  - 点击"使用指南"按钮，打开使用指南弹窗
  - 弹窗以卡片形式展示该服务的连接示例文本
  - SMB 使用指南内容包含：
    - Windows：地址栏输入 `\\192.168.1.100`，输入用户名密码
    - macOS：Finder → 前往 → 连接服务器 → 输入 `smb://192.168.1.100`
    - Linux：`sudo mount -t cifs //192.168.1.100/share /mnt/nas -o username=admin`
    - 手机端：iOS 文件 App / Android SMB 文件管理器
  - 弹窗标题跟随当前服务名称
- **约束条件**：
  - 指南内容基于当前服务的实际配置和预设 IP 生成
  - 服务未启用时，卡片上不显示"使用指南"按钮
- **可用场景**：帮助终端用户在各自设备上连接 NAS 服务。
- **涉及接口**：无（纯前端静态内容）

---

### 功能模块: 启用/禁用服务

#### 功能: 服务状态切换
- **功能描述**：启用或禁用某个服务。
- **预期行为**：
  - 点击卡片上的启用/禁用开关
  - 状态变更后，系统执行启动/停止服务操作
  - 操作完成后卡片状态标签更新
  - 失败时显示错误原因
- **约束条件**：
  - 禁用服务将导致已连接的客户端断开
  - 部分系统核心服务不可禁用
  - 服务启动失败时保持禁用状态并显示错误
- **可用场景**：按需开启服务，或在维护时临时关闭服务。
- **涉及接口**：
  - [启动/停止服务](../interface.md#启动停止服务)

## 预期交互

### 场景: 启用 SMB 共享
1. 管理员进入"服务设置"页面
2. 在"文件共享"Tab 找到 SMB 共享卡片，当前状态为"运行中"且已启用
3. 卡片显示快捷信息：共享路径 `/share/smb`、端口 `445`、工作组 `WORKGROUP`、SMBv3 已启用、访客访问禁止
4. 点击"配置"按钮，打开 SMB 配置弹窗
5. 在基本设置中将工作组改为 `OFFICE`
6. 在高级选项中关闭"广播服务发现"
7. 保存配置，卡片快捷信息同步更新

### 场景: 查看 SMB 使用指南
1. 管理员点击 SMB 共享卡片的"使用指南"
2. 弹窗显示各平台的连接方式
3. Windows 页面显示 `\\192.168.1.100` 连接地址
4. 管理员可将内容复制发送给团队成员

### 场景: 禁用 FTP 服务
1. 管理员发现 FTP 服务当前为"运行中"
2. 点击 FTP 卡片的启用开关，将其关闭
3. 服务停止，卡片变为禁用状态，显示"开启后即可配置和访问"

## 数据需求（Mock）

- **服务列表数据**：
  - 每项包含：id、name、description、icon、iconColor、enabled、category、status、url、quickInfo（标签数组）、guideContent（指南文本）、config（服务专属配置对象）
- **SMB 专属配置数据结构**：
  - `workgroup`: 工作组（字符串，默认 `WORKGROUP`）
  - `serverName`: 服务器名称（字符串，默认 `NAS-SERVER`）
  - `bindInterface`: 绑定接口（字符串，默认 `全部接口`）
  - `enableSmb1`: 启用 SMB 1.0（布尔值，默认 `false`）
  - `enableSmb2`: 启用 SMB 2.0（布尔值，默认 `true`）
  - `enableSmb3`: 启用 SMB 3.0（布尔值，默认 `true`）
  - `guestAccess`: 允许访客访问（布尔值，默认 `false`）
  - `announceService`: 广播服务发现（布尔值，默认 `true`）
- **其他服务配置占位数据结构**（同 types.ts 中定义，仅用于原型展示）：
  - HTTP 管理、AFP、FTP、WebDAV、NFS 各自有独立的 config 对象结构
