# PRD: 网卡详情

## 页面概要

| 属性 | 值 |
|------|-----|
| 路由 | `/system/device/network/:name` |
| 菜单标题 | 网卡详情（隐藏菜单） |
| 页面权限 | 已登录用户 |
| 页面类型 | 详情页（子页面） |

## 需求概述

**总体功能**：网卡详情页展示单个网络接口的详细信息，包括基础配置、实时流量图表、历史流量统计等。通过路由参数 `name` 定位具体网卡。

**目标用户**：需要排查网络问题、监控网络流量或配置网络参数的技术用户或管理员。

**核心价值**：提供网卡级别的深度信息，帮助用户监控网络性能、排查连接问题。

---

## 功能规格

### 功能模块: 基础信息

#### 功能: 网卡基础参数
- **功能描述**：展示网卡的基本配置和状态信息。
- **预期行为**：
  - 显示接口名称（如 eth0）
  - 显示 MAC 地址
  - 显示 IP 地址 / 子网掩码 / 网关
  - 显示 DNS 服务器
  - 显示连接状态（已连接/断开），颜色区分
  - 显示连接速率（如 1000 Mbps）
  - 显示双工模式（全双工/半双工）
- **约束条件**：
  - 部分信息（如连接速率）由网卡硬件和交换机协商决定
- **可用场景**：查看网卡当前配置，排查网络连接问题。
- **涉及接口**：
  - 获取网卡详情

#### 功能: 网卡配置编辑
- **功能描述**：修改网卡的网络配置参数。
- **预期行为**：
  - 点击"编辑配置"按钮，打开配置表单
  - 支持切换 DHCP / 静态 IP 模式
  - 静态 IP 模式下可编辑 IP、子网掩码、网关、DNS
  - 保存后提示重启网络服务
- **约束条件**：
  - 修改管理网口配置可能导致断开连接，需二次确认
  - 错误的配置可能导致网络不可达
- **可用场景**：修改网卡 IP 地址或切换网络模式。
- **涉及接口**：
  - 更新网卡配置

---

### 功能模块: 流量监控图表

#### 功能: 实时流量曲线
- **功能描述**：展示网卡上行/下行速率的实时曲线图。
- **预期行为**：
  - 显示上行速率和下行速率的两条曲线
  - 图表自动刷新（默认 5 秒）
  - 显示当前实时速率数值
  - 支持时间范围切换（最近1小时/24小时/7天）
- **约束条件**：
  - 数据来源于系统网络监控
  - 断开状态下速率为 0
- **可用场景**：监控网络带宽使用情况，发现异常流量。
- **涉及接口**：
  - 获取网卡流量时间序列数据

---

### 功能模块: 数据统计

#### 功能: 累计流量统计
- **功能描述**：展示网卡的累计收发数据统计。
- **预期行为**：
  - 显示总接收字节数 / 总发送字节数
  - 显示接收包数 / 发送包数
  - 显示错误包数 / 丢弃包数
  - 大数值自动格式化显示（KB/MB/GB/TB/PB）
  - 显示自开机以来的累计值
- **约束条件**：
  - 累计值在系统重启后清零（除非持久化存储）
  - 错误包/丢弃包持续增加可能表示硬件或线路问题
- **可用场景**：评估网络接口的整体使用情况，排查网络质量问题。
- **涉及接口**：
  - 获取网卡累计统计

## 预期交互

### 场景: 排查网络连接问题
1. 用户从设备管理页点击某网卡"详情"
2. 页面跳转至网卡详情页，显示 eth0 的信息
3. 基础信息显示连接状态"已连接"（绿色），速率 1000 Mbps，全双工
4. 实时流量图显示当前有数据传输，排除物理层问题
5. 累计统计中错误包数为 0，排除数据链路层问题

### 场景: 监控网络带宽使用
1. 用户查看实时流量曲线
2. 发现下行速率持续在 900 Mbps 以上，接近千兆上限
3. 切换时间范围到"最近24小时"，查看历史峰值
4. 确认需要升级网络带宽或增加网口做链路聚合

### 场景: 修改静态 IP
1. 用户点击"编辑配置"
2. 将 DHCP 模式切换为静态 IP
3. 输入新的 IP 地址 192.168.1.100、子网掩码 255.255.255.0
4. 保存后系统提示网络服务将重启
5. 页面短暂断开后恢复，显示新 IP 已生效

## 数据需求（Mock）

- **网卡基础信息数据**：接口名称、MAC、IP、子网掩码、网关、DNS、连接状态、速率、双工模式
- **实时流量时间序列数据**：上行/下行速率的时间序列
- **累计统计数据**：收发字节数、包数、错误包、丢弃包
