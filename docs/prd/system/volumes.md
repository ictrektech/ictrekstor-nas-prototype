# PRD: 存储空间管理

## 页面概要

| 属性 | 值 |
|------|-----|
| 路由 | `/system/volumes` |
| 菜单标题 | 存储空间管理 |
| 菜单图标 | `lucide:box` |

## 需求概述

存储空间管理页是 NAS 存储系统的核心管理界面，用于管理存储池和存储空间（卷）。页面展示存储池卡片、存储空间列表，并支持创建存储池、创建存储空间、配置用户权限等操作。

## 功能行为

### 1. 统计概览

- 存储空间总数
- 基于存储池的卷数量
- 基于目录的卷数量
- 总容量（GB/TB 自动转换）
- 总已用容量

### 2. 存储池卡片

每张卡片展示：
- 存储池名称、RAID 类型
- 健康状态
- 总容量 / 已用容量 / 可用容量
- 容量使用进度条
- 池内所有存储空间列表

### 3. 存储空间操作

| 操作 | 行为 |
|------|------|
| 创建存储空间 | 弹出创建存储空间弹窗，选择基于存储池或目录 |
| 重命名 | 弹出重命名弹窗 |
| 配置用户权限 | 弹出用户权限配置弹窗设置访问用户 |
| 删除 | 确认后删除存储空间 |

### 4. 创建存储池

- 弹出创建存储池弹窗
- 选择 RAID 类型
- 选择参与组池的硬盘
- 确认创建

### 5. 创建存储空间

- 基于存储池：选择已有存储池，设置名称、容量、文件系统
- 基于目录：选择本地目录路径，设置名称

## 预期交互

- 页面加载时并行获取卷、存储池、磁盘数据
- 未使用的磁盘可用于创建新存储池
- 存储池卡片可展开/收起内部卷列表
- 操作成功后自动刷新数据

## 数据需求（Mock）

- 存储空间列表数据（名称、类型、容量、文件系统、所属存储池、用户权限等）
- 存储池列表数据（名称、RAID 类型、健康状态、容量、组成磁盘等）
- 磁盘列表数据（设备名、型号、容量、使用状态等）
- 用户列表数据（用于配置存储空间访问权限）
