# PRD: 存储空间管理

## 页面概要

| 属性 | 值 |
|------|-----|
| 路由 | `/system/volumes` |
| 菜单标题 | 存储空间管理 |
| 菜单图标 | `lucide:box` |
| 页面权限 | 管理员 |
| 页面类型 | 管理页（主页面） |

## 需求概述

**总体功能**：存储空间管理页是 NAS 存储系统的核心管理界面，用于管理存储池和存储空间（卷）。页面展示存储池卡片、存储空间列表，并支持创建存储池、创建存储空间、配置用户权限等操作。

**目标用户**：NAS 系统管理员，负责存储资源的规划和管理。

**核心价值**：提供存储池和存储空间的统一管理入口，支持灵活的资源分配和权限控制。

---

## 功能规格

### 功能模块: 统计概览

#### 功能: 存储统计卡片
- **功能描述**：在页面顶部展示存储资源的关键指标。
- **预期行为**：
  - 显示存储空间总数
  - 显示基于存储池的卷数量
  - 显示基于目录的卷数量
  - 显示总容量（GB/TB 自动转换）
  - 显示总已用容量和可用容量
- **约束条件**：
  - 总容量计算需考虑 RAID 冗余开销
  - 统计数据随存储配置变化实时更新
- **可用场景**：快速了解设备存储资源的整体使用情况。
- **涉及接口**：
  - 获取存储统计摘要

---

### 功能模块: 存储池管理

#### 功能: 存储池卡片展示
- **功能描述**：以卡片形式展示所有存储池的概要信息。
- **预期行为**：
  - 每张卡片展示：存储池名称、RAID 类型
  - 显示健康状态（正常/警告/故障），颜色区分
  - 显示总容量 / 已用容量 / 可用容量
  - 显示容量使用进度条
  - 显示池内所有存储空间列表
  - 卡片可展开/收起内部卷列表
  - 点击卡片可跳转存储池详情页
- **约束条件**：
  - 存储池卡片为只读展示，修改需进入详情或执行特定操作
  - 存储池删除需先删除其下所有卷
- **可用场景**：查看各存储池的使用状况和健康状态。
- **涉及接口**：
  - 获取存储池列表

#### 功能: 创建存储池
- **功能描述**：创建新的存储池，将多块硬盘组合为 RAID。
- **预期行为**：
  - 点击"创建存储池"按钮，弹出创建弹窗
  - 第一步：选择 RAID 类型（RAID 0/1/5/6/10）
  - 第二步：选择参与组池的硬盘（从未使用硬盘中选择）
  - 显示 RAID 类型的说明和容量计算预览
  - 确认后系统开始创建存储池，显示进度
  - 创建成功后刷新存储池列表
- **约束条件**：
  - 仅管理员可创建存储池
  - 不同 RAID 类型对硬盘数量有最低要求
  - 已用于其他存储池或包含数据的硬盘不可选择
  - 创建过程耗时较长，不可中途取消
- **可用场景**：新设备安装后初始化存储，或扩展现有存储池。
- **涉及接口**：
  - 创建存储池
  - 获取未使用硬盘列表

---

### 功能模块: 存储空间（卷）管理

#### 功能: 存储空间列表
- **功能描述**：以表格或卡片形式展示所有存储空间（卷）。
- **预期行为**：
  - 展示卷名称、类型（基于存储池/基于目录）、容量、文件系统、所属存储池
  - 显示用户权限配置状态
  - 提供操作按钮：重命名、配置权限、删除
- **约束条件**：
  - 仅管理员可管理存储空间
  - 删除卷会丢失该卷内所有数据
- **可用场景**：查看和管理所有存储空间。
- **涉及接口**：
  - 获取存储空间列表

#### 功能: 创建存储空间
- **功能描述**：创建新的存储空间（卷）。
- **预期行为**：
  - 点击"创建存储空间"按钮，弹出创建弹窗
  - 选择类型：基于存储池 或 基于目录
  - **基于存储池**：选择已有存储池，设置卷名称、容量、文件系统（ext4/btrfs）
  - **基于目录**：选择本地目录路径，设置卷名称
  - 确认后创建存储空间，刷新列表
- **约束条件**：
  - 仅管理员可创建存储空间
  - 基于存储池时，容量不能超过存储池可用容量
  - 基于目录时，路径必须是系统中存在的有效目录
- **可用场景**：为用户分配独立的存储空间。
- **涉及接口**：
  - 获取本地目录列表（基于目录类型时用于选择目录路径）
  - 创建存储空间

#### 功能: 重命名存储空间
- **功能描述**：修改存储空间的名称。
- **预期行为**：
  - 点击"重命名"按钮，弹出输入弹窗
  - 输入新名称，确认后完成重命名
  - 不影响卷内数据和挂载路径
- **约束条件**：
  - 名称不能为空，不能包含特殊字符
- **可用场景**：调整存储空间的命名以更好地反映用途。
- **涉及接口**：
  - 重命名存储空间

#### 功能: 配置用户权限
- **功能描述**：配置哪些用户可以访问该存储空间及其权限。
- **预期行为**：
  - 点击"配置权限"按钮，弹出权限配置弹窗
  - 显示用户列表，可为每个用户设置权限：无/只读/读写
  - 支持按用户组批量设置权限
  - 支持设置默认权限（新用户的默认权限）
  - 保存后即时生效
- **约束条件**：
  - 至少保留一个管理员具备读写权限
  - 权限变更不影响已打开的文件会话
- **可用场景**：控制不同用户对存储空间的访问权限。
- **涉及接口**：
  - 更新存储空间权限
  - 获取用户列表（用于配置权限）

#### 功能: 删除存储空间
- **功能描述**：删除存储空间，释放其占用的容量。
- **预期行为**：
  - 点击"删除"按钮，弹出二次确认弹窗
  - 弹窗明确提示删除将丢失该卷内所有数据
  - 确认后删除存储空间
  - 刷新列表，释放的容量回到所属存储池
- **约束条件**：
  - 删除前需确认卷内无重要数据
  - 正在被访问的卷可能无法删除
  - 删除操作不可逆
- **可用场景**：清理不再需要的存储空间。
- **涉及接口**：
  - 删除存储空间

## 预期交互

### 场景: 创建存储池
1. 管理员进入"存储空间管理"页面
2. 点击"创建存储池"
3. 选择 RAID 5，系统提示至少需要 3 块硬盘
4. 从未使用硬盘中选择 3 块硬盘，显示可用容量预览
5. 确认创建，系统显示 RAID 构建进度（约数分钟）
6. 创建完成后存储池卡片出现在页面中

### 场景: 创建存储空间并分配权限
1. 管理员点击"创建存储空间"
2. 选择类型"基于存储池"，选择刚创建的存储池
3. 设置名称"项目数据"、容量 2TB、文件系统 ext4
4. 创建完成后，点击"配置权限"
5. 为"项目组"用户组设置"读写"权限，其他用户默认"无"
6. 项目组成员即可在"我的文件"中看到该存储空间

### 场景: 扩容存储空间
1. 管理员发现"备份"卷容量即将耗尽
2. 点击该卷的"编辑"按钮，调整容量从 1TB 到 2TB
3. 确认后系统扩展卷容量，数据不受影响
4. 用户空间可用容量同步增加

## 数据需求（Mock）

- **存储空间列表数据**：卷ID、名称、类型、容量、文件系统、所属存储池、用户权限配置
- **存储池列表数据**：池ID、名称、RAID类型、健康状态、总容量、已用容量、可用容量、组成磁盘列表
- **磁盘列表数据**：设备名、型号、容量、使用状态（已用/未使用）
- **用户列表数据**：用户ID、用户名、角色（用于配置存储空间访问权限）
