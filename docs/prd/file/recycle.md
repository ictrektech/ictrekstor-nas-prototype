# PRD: 回收站

## 页面概要

| 属性 | 值 |
|------|-----|
| 路由 | `/file/recycle` |
| 菜单标题 | 回收站 |
| 菜单图标 | `lucide:trash-2` |

## 需求概述

回收站页面用于管理已删除但尚未彻底删除的文件和文件夹。用户可查看、还原或永久删除回收站中的内容。按存储空间和团队文件夹分组展示。

## 功能行为

### 1. 目录树结构

- `root-space` 节点：存储空间分组
- `root-team` 节点：团队文件夹分组
- 子节点为具体的存储空间或团队文件夹
- 叶子节点为文件夹

### 2. 文件操作

| 操作 | 行为 |
|------|------|
| 还原 | 将文件还原到原位置，从回收站移除 |
| 彻底删除 | 永久删除文件，不可恢复 |
| 刷新 | 重新加载回收站数据 |

### 3. 视图模式

- 页面处于回收站模式
- 隐藏"新建文件夹"按钮
- 操作按钮显示"还原"和"删除"而非常规操作

### 4. 空状态

- 当回收站无文件时，显示"暂无回收文件"

## 预期交互

- 点击"还原"后立即将文件从列表移除，显示成功提示
- 点击"彻底删除"后文件永久移除，显示成功提示
- 导航支持返回根节点

## 数据需求（Mock）

- 回收站目录树数据（按存储空间和团队文件夹分组）
- 回收站文件列表数据（含原路径信息，用于还原）

## 关联页面

- 复用全局文件管理组件
- 回收站文件来源于"我的文件"、"公共文件"等页面的删除操作
