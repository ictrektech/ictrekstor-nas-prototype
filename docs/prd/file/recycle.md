## 需求概述

回收站页面用于管理已删除但尚未彻底删除的文件和文件夹。用户可查看、还原或永久删除回收站中的内容。采用文件管理器布局，按存储空间和团队文件夹分组展示。

## 功能行为

### 1. 页面布局

- **顶部**：`RecycleBinHeader` 统计概览（文件数、文件夹数）
- **主体**：左侧目录树 + 右侧文件管理面板

### 2. 目录树结构

- `root-space` 节点：存储空间分组
- `root-team` 节点：团队文件夹分组
- 子节点为具体的存储空间或团队文件夹
- 叶子节点为文件夹

### 3. 文件操作

| 操作 | 行为 |
|------|------|
| 还原 | 将文件还原到原位置，从回收站移除 |
| 彻底删除 | 永久删除文件，不可恢复 |
| 刷新 | 重新加载回收站数据 |

### 4. 视图模式

- `mode="recycle"`：文件管理面板处于回收站模式
- 隐藏"新建文件夹"按钮
- 操作按钮显示"还原"和"删除"而非常规操作

### 5. 空状态

- 当回收站无文件时，显示"暂无回收文件"

## 预期交互

- 点击"还原"后立即将文件从列表移除，显示成功提示
- 点击"彻底删除"后文件永久移除，显示成功提示
- 面包屑导航支持返回根节点
