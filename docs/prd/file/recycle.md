# PRD: 回收站

## 页面概要

| 属性 | 值 |
|------|-----|
| 路由 | `/file/recycle` |
| 菜单标题 | 回收站 |
| 菜单图标 | `lucide:trash-2` |
| 页面权限 | 已登录用户 |
| 页面类型 | 文件管理页（主页面） |

## 需求概述

**总体功能**：回收站页面用于管理已删除但尚未彻底删除的文件和文件夹。用户可查看、还原或永久删除回收站中的内容。按存储空间和团队文件夹分组展示。

**目标用户**：需要恢复误删文件或彻底清理已删除文件的 NAS 用户。

**核心价值**：为用户提供文件删除的安全缓冲区，防止误删导致数据丢失，同时支持彻底清理释放空间。

---

## 功能规格

### 功能模块: 回收站目录树

#### 功能: 分组目录树
- **功能描述**：以树形结构展示回收站中的文件，按来源分组。
- **预期行为**：
  - `root-space` 节点：存储空间分组，子节点为具体的存储空间
  - `root-team` 节点：团队文件夹分组，子节点为具体的团队文件夹
  - 叶子节点为被删除的文件夹
  - 点击节点加载对应回收站文件列表
- **约束条件**：
  - 仅展示当前用户有权限查看的已删除文件
  - 不同来源的文件按原始位置分组
- **可用场景**：按原始位置查找被删除的文件。
- **涉及接口**：
  - 获取回收站目录树

---

### 功能模块: 回收站文件操作

#### 功能: 文件还原
- **功能描述**：将回收站中的文件或文件夹还原到原始位置。
- **预期行为**：
  - 选中文件后点击"还原"按钮
  - 系统自动将文件恢复到删除前的原路径
  - 还原成功后文件从回收站列表移除
  - 显示成功提示
- **约束条件**：
  - 原路径所在目录需仍存在且用户具备写入权限
  - 原路径已存在同名文件时需提示覆盖或重命名
  - 还原后文件恢复原有权限设置
- **可用场景**：恢复误删的重要文件。
- **涉及接口**：
  - 还原文件到原位置

#### 功能: 彻底删除
- **功能描述**：永久删除回收站中的文件或文件夹，不可恢复。
- **预期行为**：
  - 选中文件后点击"彻底删除"按钮
  - 弹出二次确认弹窗，提示操作不可恢复
  - 确认后永久删除文件，从回收站列表移除
  - 显示成功提示
- **约束条件**：
  - 彻底删除后文件不可恢复
  - 删除文件夹会递归删除其下所有内容
  - 批量删除时列出待删除文件数量
- **可用场景**：确认不再需要某文件，彻底清理释放存储空间。
- **涉及接口**：
  - 永久删除文件

#### 功能: 刷新回收站
- **功能描述**：重新加载回收站数据。
- **预期行为**：
  - 点击"刷新"按钮，重新获取回收站目录树和文件列表
  - 刷新后保持当前选中的分组节点
- **约束条件**：无特殊约束。
- **可用场景**：在其他页面删除文件后，刷新回收站查看更新。
- **涉及接口**：
  - 重新获取回收站目录树
  - 重新获取回收站文件列表

---

### 功能模块: 回收站视图

#### 功能: 回收站专用视图
- **功能描述**：回收站模式下显示与常规文件管理不同的操作按钮和提示。
- **预期行为**：
  - 隐藏"新建文件夹"按钮
  - 隐藏"上传"按钮
  - 操作按钮显示"还原"和"彻底删除"而非常规操作
  - 文件列表显示删除时间（额外列）
- **约束条件**：回收站视图不可进行常规的文件创建和修改操作。
- **可用场景**：专注于回收站文件的还原和清理操作。
- **涉及接口**：无（纯前端视图控制）

---

### 功能模块: 空状态

#### 功能: 回收站空状态
- **功能描述**：当回收站无文件时显示空状态提示。
- **预期行为**：
  - 目录树为空时显示"暂无回收文件"提示
  - 文件列表为空时显示空状态图标和文字
- **约束条件**：空状态仅为提示，不提供操作入口。
- **可用场景**：确认回收站已清空。
- **涉及接口**：无（纯前端展示）

---

## 预期交互

### 场景: 恢复误删文件
1. 用户进入"回收站"页面
2. 左侧目录树显示"存储空间1"下有被删除的文件
3. 用户点击"存储空间1"，右侧加载该空间下的已删除文件
4. 用户选中误删的"重要文档.docx"
5. 点击"还原"按钮，文件从回收站消失
6. 系统提示"已成功还原到原位置"

### 场景: 清理回收站释放空间
1. 用户进入回收站，选中多个不再需要的文件
2. 点击"彻底删除"
3. 弹窗提示"确定永久删除选中的 5 个文件？此操作不可恢复。"
4. 用户确认，文件永久删除
5. 列表自动刷新，剩余文件继续显示
