# PRD: 设备全部文件

## 页面概要

| 属性 | 值 |
|------|-----|
| 路由 | `/file/all-files` |
| 菜单标题 | 设备全部文件 |
| 菜单图标 | `lucide:hard-drive` |
| 页面权限 | 管理员 |
| 页面类型 | 文件管理页（主页面） |

## 需求概述

**总体功能**："设备全部文件"页面提供管理员视角的全局文件浏览功能，可查看 NAS 设备上所有存储空间中的文件和文件夹，支持管理员级别的文件操作。

**目标用户**：NAS 系统管理员，需要全局监控和管理设备上所有文件。

**核心价值**：为管理员提供全局文件视图，便于审计、排查问题和进行全局文件管理。

---

## 功能规格

### 功能模块: 全局目录树

#### 功能: 全局存储空间目录树
- **功能描述**：以树形结构展示 NAS 设备上所有存储空间及文件夹层级关系。
- **预期行为**：
  - 根节点为所有存储空间（如"存储空间1"、"存储空间2"）
  - 每个存储空间下包含标准文件夹（文档、图片、视频、音乐、软件等）
  - 支持多级文件夹嵌套展开/收起
  - 点击节点加载对应文件列表
- **约束条件**：
  - 仅管理员可访问此页面
  - 展示所有存储空间，不受普通用户权限限制
- **可用场景**：管理员全局浏览设备文件结构，排查问题或进行审计。
- **涉及接口**：
  - `GET /api/file/all/tree` — 获取全局目录树

---

### 功能模块: 管理员级文件操作

#### 功能: 全局文件浏览
- **功能描述**：在右侧文件面板展示当前选中目录的文件列表。
- **预期行为**：
  - 支持列表视图和网格视图
  - 单击选中文件，双击预览/下载
  - 显示文件名、类型、大小、修改时间、所有者
- **约束条件**：
  - 浏览不受普通用户权限限制
  - 某些系统文件可能受保护不可操作
- **可用场景**：管理员查看任意存储空间中的文件。
- **涉及接口**：
  - `GET /api/file/all/list` — 获取全局文件列表

#### 功能: 新建文件夹
- **功能描述**：在任意存储空间中创建新文件夹。
- **预期行为**：
  - 点击"新建文件夹"按钮，弹出输入弹窗
  - 输入文件夹名称，确认后创建
  - 创建成功后刷新文件列表
- **约束条件**：
  - 名称不能为空，不能包含特殊字符
  - 同目录下名称不可重复
- **可用场景**：管理员为设备全局整理文件结构。
- **涉及接口**：
  - `POST /api/file/all/mkdir` — 全局创建文件夹

#### 功能: 文件重命名
- **功能描述**：修改文件或文件夹名称。
- **预期行为**：
  - 通过右键菜单或工具栏触发重命名弹窗
  - 输入新名称，确认后完成重命名
  - 刷新文件列表
- **约束条件**：
  - 新名称不能与同目录下其他文件重名
  - 系统关键文件可能受保护不可重命名
- **可用场景**：管理员修正全局文件命名。
- **涉及接口**：
  - `PUT /api/file/all/rename` — 全局重命名

#### 功能: 文件搜索
- **功能描述**：按文件名在全局范围搜索文件。
- **预期行为**：
  - 搜索框输入关键词，回车触发搜索
  - 搜索结果跨存储空间展示
  - 显示文件所属存储空间信息
- **约束条件**：
  - 搜索范围覆盖所有存储空间
  - 大容量设备搜索可能需要较长时间
- **可用场景**：管理员查找特定文件在设备中的位置。
- **涉及接口**：
  - `GET /api/file/all/search` — 全局搜索文件

#### 功能: 文件下载
- **功能描述**：将文件下载到本地。
- **预期行为**：
  - 选中文件后点击"下载"按钮
  - 浏览器触发文件下载
  - 支持批量下载
- **约束条件**：
  - 管理员具备所有文件的读取权限
- **可用场景**：管理员取回设备上的文件到本地。
- **涉及接口**：
  - `GET /api/file/download` — 下载文件
  - `POST /api/file/download/batch` — 批量下载

---

### 功能模块: 跨存储空间导航

#### 功能: 导航支持跨存储空间跳转
- **功能描述**：路径导航支持在不同存储空间之间跳转。
- **预期行为**：
  - 面包屑导航显示完整路径，包含存储空间名称
  - 点击面包屑中的存储空间节点可跳转到对应空间的根目录
  - 不同存储空间的文件列表独立加载
- **约束条件**：
  - 跨存储空间操作需考虑性能影响
- **可用场景**：管理员在不同存储空间之间快速切换查看。
- **涉及接口**：无（纯前端路由和状态管理）

## 预期交互

### 场景: 全局文件审计
1. 管理员进入"设备全部文件"页面
2. 左侧目录树加载所有存储空间
3. 管理员点击"存储空间2"，右侧加载该空间根目录文件
4. 管理员使用搜索框输入"机密"，跨存储空间搜索相关文件
5. 搜索结果展示匹配文件及其所属存储空间

### 场景: 全局文件整理
1. 管理员在"存储空间1"的根目录发现命名不规范的文件夹
2. 使用重命名功能修正文件夹名称
3. 在"存储空间2"下新建"归档资料"文件夹
4. 整理完成后刷新目录树查看全局结构

---

## 数据需求（Mock）

- **全局存储空间目录树数据**：所有存储空间及文件夹层级结构
- **全局文件列表数据**：文件名、类型、大小、修改时间、所有者、所属存储空间
- **各存储空间基本信息**：名称、总容量、已用容量
