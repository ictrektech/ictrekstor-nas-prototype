# PRD: 存储池详情

## 页面概要

| 属性 | 值 |
|------|-----|
| 路由 | `/storage/pools/detail/:id` |
| 菜单标题 | 存储池详情（隐藏菜单） |
| 页面权限 | 已登录用户 |
| 页面类型 | 详情页（子页面） |

## 需求概述

**总体功能**：存储池详情页展示单个存储池的全面信息，包括基本信息、性能监控、磁盘组成、容量分配、操作日志等。通过 Tab 页签组织不同维度的信息。

**目标用户**：需要深入了解存储池运行状态和进行管理操作的技术用户或管理员。

**核心价值**：提供存储池级别的深度信息，帮助用户监控存储性能、排查磁盘故障、优化容量分配。

---

## 功能规格

### 功能模块: 页面头部

#### 功能: 存储池头部信息
- **功能描述**：页面顶部展示存储池的核心标识信息和状态。
- **预期行为**：
  - 显示返回按钮（返回存储空间管理页）
  - 显示存储池名称
  - 显示 RAID 类型、硬盘数量
  - 显示状态标签（正常/警告），颜色区分
  - 显示总容量、已用容量、健康度
- **约束条件**：
  - 状态基于存储池内磁盘 SMART 和 RAID 状态综合评估
- **可用场景**：快速确认存储池身份和当前健康状态。
- **涉及接口**：
  - `GET /api/storage/pools/:id` — 获取存储池基本信息

---

### 功能模块: 性能监控 Tab

#### 功能: 读写速率趋势
- **功能描述**：展示存储池读写速率的历史趋势图。
- **预期行为**：
  - 显示读取速率和写入速率的曲线
  - 支持时间范围选择（最近1小时/24小时/7天/30天）
  - 显示平均/峰值速率
  - 图表自动刷新
- **约束条件**：
  - 数据来源于存储池 I/O 监控
- **可用场景**：分析存储池 I/O 性能负载。
- **涉及接口**：
  - `GET /api/storage/pools/:id/metrics/io` — 获取存储池 I/O 时间序列数据

#### 功能: IOPS 趋势
- **功能描述**：展示存储池每秒 I/O 操作数的历史趋势。
- **预期行为**：
  - 显示读 IOPS 和写 IOPS 的曲线
  - 支持时间范围选择
  - 显示平均/峰值 IOPS
- **约束条件**：
  - 与读写速率趋势同步刷新
- **可用场景**：评估存储池响应能力。
- **涉及接口**：
  - `GET /api/storage/pools/:id/metrics/iops` — 获取存储池 IOPS 时间序列数据

#### 功能: 延迟趋势
- **功能描述**：展示存储池 I/O 延迟的历史趋势。
- **预期行为**：
  - 显示读延迟和写延迟的曲线
  - 支持时间范围选择
  - 延迟过高时曲线区域变红
- **约束条件**：
  - 延迟数据受系统负载影响，波动较大
- **可用场景**：发现存储性能瓶颈，判断是否需要优化。
- **涉及接口**：
  - `GET /api/storage/pools/:id/metrics/latency` — 获取存储池延迟时间序列数据

---

### 功能模块: 磁盘组成 Tab

#### 功能: 池内磁盘列表
- **功能描述**：展示组成该存储池的所有物理磁盘。
- **预期行为**：
  - 表格展示每块磁盘：盘位号、设备名、型号、容量、健康状态、温度
  - 磁盘故障时行显示红色高亮和告警图标
  - 点击磁盘行可跳转至磁盘详情页
  - 支持按状态筛选（正常/警告/故障）
- **约束条件**：
  - 磁盘列表为只读展示，不可在此页面移除或添加磁盘
  - 存储池运行期间不可更换磁盘（需进入维护模式）
- **可用场景**：查看存储池的磁盘组成，定位故障磁盘。
- **涉及接口**：
  - `GET /api/storage/pools/:id/disks` — 获取存储池内磁盘列表

#### 功能: 磁盘故障告警
- **功能描述**：当池内磁盘出现故障时，显示告警提示。
- **预期行为**：
  - 磁盘故障时页面顶部显示告警横幅
  - 告警信息包含故障磁盘名称和建议操作
  - 告警横幅提供跳转磁盘详情的链接
- **约束条件**：
  - 告警基于 SMART 和 RAID 状态判断
- **可用场景**：及时发现存储池磁盘故障，采取维护措施。
- **涉及接口**：
  - `GET /api/storage/pools/:id/alerts` — 获取存储池告警信息

---

### 功能模块: 容量分配 Tab

#### 功能: 容量分布可视化
- **功能描述**：以图表形式展示存储池总容量的分配情况。
- **预期行为**：
  - 显示存储池总容量
  - 以饼图或堆叠条形图展示各存储空间/卷占用的容量
  - 显示剩余可用容量
  - 点击某卷可查看详情
- **约束条件**：
  - 容量数据以存储池实际可用容量为准（考虑 RAID 冗余开销）
- **可用场景**：了解存储池容量在各卷之间的分配情况。
- **涉及接口**：
  - `GET /api/storage/pools/:id/capacity` — 获取存储池容量分配数据

#### 功能: 容量使用趋势
- **功能描述**：展示存储池容量使用量的历史趋势。
- **预期行为**：
  - 显示已用容量随时间变化的趋势曲线
  - 支持时间范围选择
  - 预测容量耗尽时间（基于历史增长速度）
- **约束条件**：
  - 预测数据仅供参考，受实际使用情况影响
- **可用场景**：规划存储扩容，预判容量不足风险。
- **涉及接口**：
  - `GET /api/storage/pools/:id/capacity/trend` — 获取容量使用趋势数据

---

### 功能模块: 操作日志 Tab

#### 功能: 存储池操作日志
- **功能描述**：展示与该存储池相关的操作历史记录。
- **预期行为**：
  - 表格展示操作记录：时间、操作类型、操作人、结果、详情
  - 支持按时间范围筛选
  - 支持按操作类型筛选（创建/扩容/修复/删除等）
  - 支持分页展示
- **约束条件**：
  - 仅展示系统记录的操作日志
  - 日志保留周期受系统配置限制
- **可用场景**：审计存储池的变更历史，排查问题原因。
- **涉及接口**：
  - `GET /api/storage/pools/:id/logs` — 获取存储池操作日志

## 预期交互

### 场景: 检查存储池健康
1. 用户从存储空间管理页点击某存储池"详情"
2. 页面跳转至存储池详情页，默认显示性能监控 Tab
3. 头部显示存储池"Pool-1"，RAID 5，6 块盘，状态正常
4. 用户切换到"磁盘组成"Tab，查看各磁盘状态均正常
5. 温度列显示所有磁盘温度在 35-45°C 范围内

### 场景: 分析容量分配
1. 用户切换到"容量分配"Tab
2. 饼图显示总容量 10TB，卷"数据"占用 6TB，卷"备份"占用 2TB，剩余 2TB
3. 趋势图显示容量使用量在过去30天内增长了 500GB
4. 系统预测按当前增速约 4 个月后容量耗尽

### 场景: 排查磁盘故障
1. 页面顶部出现红色告警横幅"磁盘 /dev/sdc 状态异常"
2. 用户切换到"磁盘组成"Tab，发现 /dev/sdc 行显示红色
3. 点击该行跳转至磁盘详情页查看 SMART 详情
4. 返回后查看"操作日志"，确认最近是否有相关维护操作

---

## 数据需求（Mock）

- **存储池基本信息数据**：名称、RAID类型、硬盘数量、状态、总容量、已用容量
- **性能监控时间序列数据**：读写速率、IOPS、延迟的时间序列
- **磁盘组成数据**：池内各磁盘的盘位号、设备名、型号、容量、状态、温度
- **容量分配数据**：各卷占用的容量分布、使用趋势、预测数据
- **操作日志数据**：时间、操作类型、操作人、结果、详情描述
