## Context

当前项目基于 Vben Admin 5.x 构建，使用 Vue 3 + Vite + TypeScript + Ant Design Vue 技术栈。生产环境 VIVIBIT NAS 系统同样基于 Vben 框架，侧边栏菜单结构如下：

- **首页**：AI 智能助手、快捷入口、存储/系统空间概览、硬件信息、网络信息
- **文件管理**：存储空间、回收站、外部设备、外链管理
- **存储管理**：硬盘管理、存储池管理、存储空间管理、公共目录管理
- **系统设置**：设备信息、网络管理、用户管理

当前原型项目侧边栏仍为 Vben 默认演示菜单（概览→分析页/工作台、演示、项目链接、关于），需要全面替换为生产环境的 NAS 功能菜单。

## Goals / Non-Goals

**Goals:**
- 将侧边栏菜单完全替换为生产环境的四大模块结构
- 为每个菜单项创建对应的页面组件原型（允许占位实现）
- 基于生产环境真实数据设计 Mock API 接口和数据结构
- 清理现有冗余的演示路由和页面
- 保持框架原有布局和主题系统不变

**Non-Goals:**
- 不修改登录认证逻辑和权限系统（保持框架默认的静态权限模式）
- 不调整主题颜色和视觉风格（与生产略有差异，用户明确说明不关心）
- 不实现真实后端接口调用（纯 Mock 数据）
- 不要求各页面的完整功能交互（原型级别，重点展示结构和数据）

## Decisions

### 1. 路由组织方式：按功能模块拆分路由文件
- **决策**：每个一级菜单对应一个独立的路由模块文件（`home.ts`、`file.ts`、`storage.ts`、`system.ts`），存放于 `router/routes/modules/`
- **理由**：与 Vben 框架的动态路由加载机制（`import.meta.glob`）保持一致，便于维护；生产环境页面 URL 路径为 `#/dashboard`、`#/fileNew/library`、`#/storage/diskManagement`、`#/system/deviceInformation`，原型中采用相似路径命名
- **替代方案**：单文件集中定义所有路由 → 拒绝，不利于后续模块扩展

### 2. 页面组件存放：按模块划分子目录
- **决策**：页面组件存放于 `views/home/`、`views/file/`、`views/storage/`、`views/system/`，每个子目录下按子页面再细分
- **理由**：与路由模块一一对应，清晰直观；符合项目规则中"按功能模块划分子目录"的约定
- **替代方案**：扁平化存放所有页面 → 拒绝，不利于大型项目维护

### 3. Mock 数据实现方式：使用 Vben 内置的 Nitro Mock 服务
- **决策**：在 `apps/backend-mock/` 或利用 Vben 的 `createMockRequest` 工具创建 Mock API
- **理由**：Vben 框架已内置 Mock 支持（基于 Nitro），无需额外引入 mockjs 等库；Mock 数据可直接写在 API 定义文件中
- **替代方案**：引入 mockjs → 拒绝，增加不必要的依赖；使用硬编码数据在组件中 → 拒绝，不符合数据层分离原则

### 4. 国际化策略：中文为主，补充英文占位
- **决策**：在 `locales/langs/zh-CN/page.json` 中添加各菜单和页面标题的中文翻译，英文语言包同步添加英文占位
- **理由**：项目规则要求中文为主；Vben 的 `$t` 机制要求所有菜单标题通过国际化键引用
- **替代方案**：硬编码中文标题 → 拒绝，不符合 Vben 框架的国际化设计

### 5. 清理策略：删除冗余而非注释
- **决策**：直接删除不再需要的 `dashboard.ts`、`demos.ts`、`vben.ts` 路由文件及对应的 `views/` 子目录
- **理由**：Git 可以追踪删除操作，保持代码库干净；原型阶段不需要保留演示代码的备份
- **替代方案**：保留文件但移除路由引用 → 拒绝，会留下无用代码

## Risks / Trade-offs

| Risk | Mitigation |
|------|-----------|
| 删除现有路由后，某些内置组件（如 `WorkbenchHeader`、`AnalysisOverview`）的引用会失效 | 首页原型不使用这些 Vben 内置的演示组件，自行构建简单的信息展示卡片；如确需使用则保留对应视图文件 |
| Mock 数据与生产实际数据不完全一致，可能导致后续对接偏差 | 在 Mock 接口注释中标注数据来源（生产环境截图/页面结构分析），并说明为原型数据 |
| 一次性变更文件较多，可能引入遗漏的路由引用错误 | 变更完成后执行 `pnpm run dev:antd` 验证，检查浏览器控制台是否有路由或组件加载报错 |
| 用户管理页面涉及权限相关 UI，与当前静态权限模式有冲突 | 用户管理页面仅展示用户列表和基本信息编辑表单，不涉及角色/权限分配等复杂逻辑 |
