# 修复 Vue 组件分拆后的页面回归问题

## 背景

刚刚完成的 `vue-component-split` 变更对项目中超大型 Vue 单体文件（>500 行）进行了分拆重构。分拆完成后，通过截图对比发现部分页面的视觉效果和功能与分拆前存在明显差异。

## 问题概述

分拆前代码已单独检出到 `D:\Projects\ictrekstor-nas-prototype-ori`，运行在 `http://localhost:5888/`；
当前项目运行在 `http://localhost:5666/`。
通过逐一截图对比，发现以下 5 个页面存在回归问题：

| 页面 | URL | 问题描述 |
|------|-----|----------|
| 文件管理 - 我的文件 | /file/my-files | ① 面包屑合并为纯文本（原：分段可点击）；② 新增"操作"列；③ 底部出现意外树状视图 |
| 文件管理 - 网络访问 | /file/network | ① 协议图标颜色丢失；② 协议名称/内容变化；③ 出现重复协议卡片 |
| 系统设置 - 设备管理 | /system/device | ① 前面板硬盘槽位示意图消失；② CPU/内存卡片样式变化；③ 硬盘状态全变为"未知" |
| 系统设置 - 存储空间管理 | /system/volumes | 页面完全空白（白屏） |
| 公共文件管理 | /system/public-files | 面包屑合并为纯文本，缺少"操作"列 |

## 根因分析

组件分拆过程中，以下操作可能导致代码丢失或错误：

1. **mock 数据提取不完整**：将硬编码数据提取到 `components/mockData.ts` 时，可能遗漏了部分数据项
2. **组件替换错误**：用通用组件（如 `FileManagerPanel`）替换了原内联实现，但通用组件行为与原始定制代码不一致
3. **样式代码丢失**：分拆时只提取了 script 逻辑，遗漏了 template 中的样式类或 scoped style
4. **路由/页面入口被覆盖**：某些页面可能在分拆过程中被完全重写，导致原始内容丢失

## 目标

将上述 5 个页面的视觉效果和功能恢复到与分拆前一致（以 `localhost:5888` 为准），同时保持组件分拆后的文件结构。

## 范围

- **在范围内**：修复 5 个页面的视觉回归问题
- **不在范围内**：新增功能、重新设计 UI、调整非相关页面

## 验收标准

- [ ] 5 个页面的截图与原始项目对比无可见差异
- [ ] 各页面交互功能正常（点击、弹窗、切换等）
- [ ] 热加载服务无报错
