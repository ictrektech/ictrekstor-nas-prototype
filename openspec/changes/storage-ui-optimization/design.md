## Context

当前项目基于 Vben Admin 5.x 构建，使用 Vue 3 + Vite + TypeScript + Ant Design Vue + Tailwind CSS 技术栈。存储管理模块包含硬盘管理、存储池列表、存储空间管理、设备全部文件等页面。

通过浏览器截图评估，当前页面的视觉风格偏向基础 Ant Design 默认样式，缺乏现代感和品牌感。本次优化以"专业、现代、友好"为目标，在不改变页面功能逻辑的前提下，全面提升视觉表现。

## Goals / Non-Goals

**Goals:**
- 提升存储页面的视觉现代感和专业感
- 优化信息层次，使关键数据更突出
- 统一存储模块的视觉风格（配色、卡片、图标、动效）
- 补全缺失的页面（存储池详情、团队文件、公共目录）
- 保持原型级别，允许占位实现和模拟数据

**Non-Goals:**
- 不改变现有页面功能逻辑和交互流程
- 不修改后端 API 真实实现（仅调整 Mock 数据）
- 不要求完全响应式适配（以桌面端展示为主）
- 不引入额外的 UI 组件库或重量级依赖

## Decisions

### 1. 视觉风格：现代卡片 + 渐变点缀
- **决策**：采用白色卡片 + 微阴影 + 圆角为基础，关键卡片头部使用品牌色渐变背景点缀
- **理由**：渐变头部能瞬间提升页面质感，白色主体保持内容可读性；与 Vben 的 shadcn-vue 风格一致
- **替代方案**：纯扁平设计 → 拒绝，缺乏层次；深色主题 → 拒绝，与当前系统整体风格不一致

### 2. 图标方案：Iconify 专业图标 + 状态着色
- **决策**：使用 `@vben/icons` 提供的 Iconify 图标替换所有 emoji 图标，按设备类型和状态动态着色
- **理由**：emoji 图标（💾、📂）不够专业；Iconify 提供大量高质量的硬件/存储相关 SVG 图标；状态着色能快速传达设备健康信息
- **替代方案**：自定义 SVG 图标 → 拒绝，开发成本高；继续使用 emoji → 拒绝，影响专业感

### 3. 状态色彩语义化
- **决策**：建立统一的状态色彩映射：正常/健康=品牌蓝/绿、警告=橙色、危险=红色、未使用/离线=灰色
- **理由**：色彩语义化能使用户快速理解设备状态，减少认知负担；与 Ant Design 的 Tag color 体系兼容
- **替代方案**：单色状态 → 拒绝，信息区分度低；过多颜色 → 拒绝，造成视觉混乱

### 4. 容量可视化：分段颜色进度条
- **决策**：容量进度条根据使用率分段变色：<70% 蓝色、70%-90% 橙色、>90% 红色
- **理由**：直观传达容量告警信息；与首页仪表盘容量展示风格保持一致
- **替代方案**：纯色进度条 → 拒绝，缺乏告警视觉提示

### 5. 卡片动效：悬浮抬升 + 边框高亮
- **决策**：卡片悬浮时增加 `translateY(-2px)` 微抬升 + 阴影加深 + 边框颜色变为品牌色
- **理由**：微动效能提升交互反馈感，让页面"活"起来；参数微小不会影响性能
- **替代方案**：无动效 → 拒绝，页面会显得呆板；复杂动画 → 拒绝，原型阶段不必要

### 6. 布局重构：硬盘管理从列表到复合卡片
- **决策**：硬盘管理页从当前横向长条卡片改为"左侧设备信息 + 右侧详情"的复合卡片，增加网格间距
- **理由**：当前横向列表在宽屏下显得过于细长，复合卡片能更好地利用空间；分区信息用网格卡片展示更清晰
- **替代方案**：保持现有列表 → 拒绝，视觉表现差；完全改为表格 → 拒绝，卡片更适合设备展示

### 7. 缺失页面补全策略：框架先行 + 数据填充
- **决策**：存储池详情页填充完整的监控图表、硬盘列表、日志等数据；团队文件和公共目录先做基础列表框架
- **理由**：存储池详情是核心页面，需要完整展示；团队文件和公共目录在原型阶段只需基础结构
- **替代方案**：全部页面完整实现 → 拒绝，超出原型范围；仅做占位文字 → 拒绝，评审体验差

## Risks / Trade-offs

| Risk | Mitigation |
|------|-----------|
| 过度设计导致代码量增加，影响原型迭代速度 | 优先使用 Tailwind CSS 工具类，避免大量自定义 SCSS；保持样式内联于组件 |
| 图标变更后某些设备类型无对应图标 | 准备通用 fallback 图标（如 `lucide:hard-drive`），确保所有类型都有显示 |
| 渐变和阴影在深色主题下显示异常 | 使用 CSS 变量控制颜色，确保深浅色主题切换时渐变适配；原型阶段以浅色主题为主 |
| 存储池详情页图表依赖 vue-echarts | 确认 echarts 已安装（当前 package.json 已包含），图表使用简洁的折线图即可 |
| 补全页面后路由冲突 | 仔细检查 `storage.ts` 路由配置，确保新旧路由不冲突 |
